 network working group john m mcquillan request for comments bolt beranek and newman nic october categories b updates rfc scheduled imp software releases this note is a reminder that the procedures suggested in rfc for scheduling imp software maintenance have now been in effect for a period of about two months two points apparently require re emphasis we reserve the hour from to a m eastern time every tuesday as the time when imps can be reloaded we will probably not use this period every tuesday but we do reserve this period every tuesday the software maintenance time period is in addition to the hardware preventive maintenance scheduled at each site we now send an imp going down message to all the hosts at the site which contains information on the time that the imp is going down the expected duration and the reason we try to send out this message an hour before soft ware releases and five to ten minutes before scheduled pre ventive maintenance further we will also notify the hosts at a site whenever possible when it is necessary to stop the imp on an unscheduled basis finally seconds before the page imp goes down a second imp going down message will be sent to the hosts informing them that the imp is going down immediately the format of the imp going down message is as follows bits all zero bits bits all zero bits coded as follows all bits zero going down in seconds bits scheduled hardware pm bits scheduled software reload bits emergency reload on restart bits how soon the imp is going down in minute units bits how long the imp will be down in minute units this rfc was put into machine readable form for entry into the online rfc archives by bbn corp under the direction of alex mckenzie page
