network working group g gregg request for comments ucsb october transmission of multiple control messages the network program at a site must be prepared to send or receive more than one control message in a single control communication during the time that the control link is blocked awaiting rfnm it is possible that a number of control messages to a particular remote host will accumulate when the link is unblocked these should all be sent in a single communication for greatest efficiency for receiving it will be necessary to have a loop which returns to decode and take action on successive control messages until the control communication is exhausted
