<div id="noteTA-f3e2a619" class="noteTA"><div class="noteTA-group"><div data-noteta-group-source="module" data-noteta-group="IT"></div></div></div>
<div role="note" class="hatnote navigation-not-searchable"><span typeof="mw:File"><a href="/wiki/Wikipedia:%E6%B6%88%E6%AD%A7%E4%B9%89" title="Wikipedia:消歧义"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/f/fb/Confusion_grey.svg/24px-Confusion_grey.svg.png" decoding="async" width="24" height="18" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/fb/Confusion_grey.svg/36px-Confusion_grey.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/f/fb/Confusion_grey.svg/48px-Confusion_grey.svg.png 2x" data-file-width="260" data-file-height="200" /></a></span><style data-mw-deduplicate="TemplateStyles:r74069148">body:not(.skin-minerva) .mw-parser-output .ifmobile>.mobile{display:none}body.skin-minerva .mw-parser-output .ifmobile>.nomobile{display:inherit;display:initial}</style><span class="ifmobile"><span class="nomobile">&#160;&#160;</span><span class="mobile"></span></span><b>提示</b>：此条目页的主题不是<b><a href="/wiki/%E4%BD%93%E7%B3%BB%E7%BB%93%E6%9E%84" class="mw-redirect" title="体系结构">体系结构</a></b>。</div> 
<p><b>指令集架构</b>（英语：<span lang="en">Instruction Set Architecture</span>，缩写为ISA），又称<b>指令集</b>或<b>指令集体系</b>，是<a href="/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E4%BD%93%E7%B3%BB%E7%BB%93%E6%9E%84" class="mw-redirect" title="计算机体系结构">计算机体系结构</a>中与<a href="/wiki/%E7%A8%8B%E5%BA%8F%E8%A8%AD%E8%A8%88" class="mw-redirect" title="程序设计">程序设计</a>有关的部分，包含了<a href="/wiki/%E8%B3%87%E6%96%99%E5%9E%8B%E5%88%A5" class="mw-redirect" title="资料类型">基本数据类型</a>，指令集，<a href="/wiki/%E5%AF%84%E5%AD%98%E5%99%A8" title="寄存器">寄存器</a>，<a href="/wiki/%E5%AF%BB%E5%9D%80%E6%A8%A1%E5%BC%8F" title="寻址模式">寻址模式</a>，<a href="/w/index.php?title=%E5%AD%98%E5%82%A8%E4%BD%93%E7%B3%BB&amp;action=edit&amp;redlink=1" class="new" title="存储体系（页面不存在）">存储体系</a>，<a href="/wiki/%E4%B8%AD%E6%96%B7" title="中断">中断</a>，<a href="/wiki/%E5%BC%82%E5%B8%B8%E5%A4%84%E7%90%86" title="异常处理">异常处理</a>以及外部<a href="/wiki/I/O" title="I/O">I/O</a>。指令集架构包含一系列的<a href="/w/index.php?title=Opcode&amp;action=edit&amp;redlink=1" class="new" title="Opcode（页面不存在）">opcode</a>即操作码（<a href="/wiki/%E6%A9%9F%E5%99%A8%E8%AA%9E%E8%A8%80" class="mw-redirect" title="机器语言">机器语言</a>），以及由特定处理器执行的基本命令。
</p><p>不同的处理器“家族”——例如<a href="/wiki/Intel" class="mw-redirect" title="Intel">Intel</a> <a href="/wiki/IA-32" title="IA-32">IA-32</a>和<a href="/wiki/X86-64" title="X86-64">x86-64</a>、<a href="/wiki/IBM" title="IBM">IBM</a>/<a href="/wiki/%E9%A3%9E%E6%80%9D%E5%8D%A1%E5%B0%94" title="飞思卡尔">Freescale</a> Power和<a href="/wiki/ARM" class="mw-redirect" title="ARM">ARM</a>处理器家族——有不同的指令集架构。<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup>
</p><p>指令集体系与<a href="/wiki/%E5%BE%AE%E6%9E%B6%E6%A7%8B" title="微架构">微架构</a>（一套用于执行指令集的微处理器设计方法）不同。使用不同微架构的电脑可以共享一种指令集。例如，<a href="/wiki/%E8%8B%B1%E7%89%B9%E7%88%BE" class="mw-redirect" title="英特尔">Intel</a>的<a href="/wiki/%E5%A5%94%E9%A8%B0" title="奔腾">Pentium</a>和<a href="/wiki/%E8%B6%85%E5%BE%AE%E5%8D%8A%E5%AF%BC%E4%BD%93" class="mw-redirect" title="超微半导体">AMD</a>的<a href="/wiki/AMD_Athlon" title="AMD Athlon">AMD Athlon</a>，两者几乎采用相同版本的<a href="/wiki/X86" title="X86">x86</a>指令集体系，但是两者在内部设计上有本质的区别。
</p><p>一些虚拟机支持基于<a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a>，<a href="/wiki/Java%E8%99%9B%E6%93%AC%E6%A9%9F" class="mw-redirect" title="Java虚拟机">Java虚拟机</a>，微软的<a href="/wiki/%E5%85%AC%E5%85%B1%E8%AF%AD%E8%A8%80%E8%BF%90%E8%A1%8C%E6%97%B6" class="mw-redirect" title="公共语言运行时">公共语言运行时</a>虚拟机所生成的<a href="/wiki/%E5%AD%97%E8%8A%82%E7%A0%81" title="字节码">字节码</a>，他们的指令集体系将bytecode（字节码）从作为一般手段的代码路径翻译成本地的机器语言，并通过解译执行并不常用的代码路径，<a href="/wiki/%E5%85%A8%E7%BE%8E%E9%81%94" title="全美达">全美达</a>以相同的方式开发了基于x86指令体系的<a href="/wiki/VLIW" class="mw-redirect" title="VLIW">VLIW</a>处理器。
</p>

<h2><span id=".E6.8C.87.E4.BB.A4.E9.9B.86.E7.9A.84.E5.88.86.E7.B1.BB"></span><span class="mw-headline" id="指令集的分类">指令集的分类</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=1" title="编辑章节：指令集的分类"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p><a href="/wiki/%E5%A4%8D%E6%9D%82%E6%8C%87%E4%BB%A4%E9%9B%86%E8%AE%A1%E7%AE%97%E6%9C%BA" class="mw-redirect" title="复杂指令集计算机">复杂指令集计算机</a>包含许多应用程序中很少使用的特定指令，由此产生的缺陷是指令长度不固定。<a href="/wiki/%E7%B2%BE%E7%AE%80%E6%8C%87%E4%BB%A4%E9%9B%86%E8%AE%A1%E7%AE%97%E6%9C%BA" title="精简指令集计算机">精简指令集计算机</a>通过只执行在程序中经常使用的指令来简化处理器的结构，而特殊操作则以子程序的方式实现，它们的特殊使用通过处理器额外的执行时间来弥补。理论上的重要类型还包括<span class="ilh-all" data-orig-title="最小指令集计算机" data-lang-code="en" data-lang-name="英语" data-foreign-title="Minimal instruction set computer"><span class="ilh-page"><a href="/w/index.php?title=%E6%9C%80%E5%B0%8F%E6%8C%87%E4%BB%A4%E9%9B%86%E8%AE%A1%E7%AE%97%E6%9C%BA&amp;action=edit&amp;redlink=1" class="new" title="最小指令集计算机（页面不存在）">最小指令集计算机</a></span><span class="noprint ilh-comment">（<span class="ilh-lang">英语</span><span class="ilh-colon">：</span><span class="ilh-link"><a href="https://en.wikipedia.org/wiki/Minimal_instruction_set_computer" class="extiw" title="en:Minimal instruction set computer"><span lang="en" dir="auto">Minimal instruction set computer</span></a></span>）</span></span>与<a href="/wiki/%E5%96%AE%E4%B8%80%E6%8C%87%E4%BB%A4%E9%9B%86" class="mw-redirect" title="单一指令集">单指令集计算机</a>，但都未用作商业处理器。另外一种派生类型是<a href="/wiki/%E8%B6%85%E9%95%BF%E6%8C%87%E4%BB%A4%E5%AD%97" title="超长指令字">超长指令字</a>，处理器接受许多经过编码的指令并通过检索提取出一个指令字并执行。
</p>
<h2><span id=".E6.A9.9F.E5.99.A8.E8.AA.9E.E8.A8.80"></span><span class="mw-headline" id="機器語言">机器语言</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=2" title="编辑章节：机器语言"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>机器语言是由<i>声明</i>和<i>指令</i>所组成的。在处理结构上，一个特定指令指明了以下几个部分：
</p>
<ul><li>用于算术运算，寻址或者控制功能的特定<a href="/wiki/%E5%AF%84%E5%AD%98%E5%99%A8" title="寄存器">寄存器</a>；</li>
<li>特定存储空间的地址或偏移量；</li>
<li>用于解译操作数的特定<a href="/wiki/%E5%AF%BB%E5%9D%80%E6%A8%A1%E5%BC%8F" title="寻址模式">寻址模式</a>；</li></ul>
<p>复杂的操作可以借由将简单的指令合并而达成，可以（在<a href="/wiki/%E5%86%AF%C2%B7%E8%AF%BA%E4%BE%9D%E6%9B%BC%E4%BD%93%E7%B3%BB" class="mw-redirect" title="冯·诺依曼体系">冯·诺依曼体系</a>中）连续的执行，也可以藉<a href="/wiki/%E6%8E%A7%E5%88%B6%E6%B5%81" class="mw-redirect" title="控制流">控制流</a>来执行指令。
</p>
<h3><span id=".E6.8C.87.E4.BB.A4.E7.B1.BB.E5.9E.8B"></span><span class="mw-headline" id="指令类型">指令类型</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=3" title="编辑章节：指令类型"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>有效的指令操作须包含：
</p>
<ul><li>数据处理与存储操作
<ul><li>将<a href="/wiki/%E6%9A%AB%E5%AD%98%E5%99%A8" class="mw-redirect" title="寄存器">寄存器</a>的值（在中央处理器作为高速缓存的存储空间）设为固定值；</li>
<li>将数据从存储空间中传送至寄存器，反之亦然。用于将数据取出并执行计算，或者将计算结果予以保存；</li>
<li>从硬件设备读取或写入数据。</li></ul></li>
<li><a href="/wiki/%E7%AE%97%E6%9C%AF%E9%80%BB%E8%BE%91%E5%8D%95%E5%85%83" class="mw-redirect" title="算术逻辑单元">算术逻辑单元</a>
<ul><li>对两个存储于寄存器的数字进行<b>add</b>，<b>subtract</b>，<b>multiply</b>，<b>divide</b>,将结果放到一个寄存器内，一个或是更多的<a href="/wiki/%E6%97%97%E6%A8%99" title="旗标">状态码</a>可能被设置在<span class="ilh-all" data-orig-title="狀態暫存器" data-lang-code="en" data-lang-name="英语" data-foreign-title="Status register"><span class="ilh-page"><a href="/w/index.php?title=%E7%8B%80%E6%85%8B%E6%9A%AB%E5%AD%98%E5%99%A8&amp;action=edit&amp;redlink=1" class="new" title="状态寄存器（页面不存在）">状态寄存器</a></span><span class="noprint ilh-comment">（<span class="ilh-lang">英语</span><span class="ilh-colon">：</span><span class="ilh-link"><a href="https://en.wikipedia.org/wiki/Status_register" class="extiw" title="en:Status register"><span lang="en" dir="auto">Status register</span></a></span>）</span></span>中；</li>
<li>执行<a href="/wiki/%E4%BD%8D%E6%93%8D%E4%BD%9C" title="位操作">位操作</a>，藉对两组数字（为两串的数字，都由零与一构成，分别存储于两个寄存器内）执行<b><a href="/wiki/%E9%82%8F%E8%BC%AF%E8%88%87" class="mw-redirect" title="逻辑与">逻辑与</a></b>和<b><a href="/wiki/%E9%80%BB%E8%BE%91%E6%88%96" title="逻辑或">逻辑或</a></b>，或者对寄存器的每一位执行<b><a href="/wiki/%E9%82%8F%E8%BC%AF%E9%9D%9E" class="mw-redirect" title="逻辑非">逻辑非</a></b>操作；</li>
<li><b>比较</b>两个寄存器中的数据（例如是大于或者相等）；</li></ul></li>
<li><a href="/wiki/%E6%8E%A7%E5%88%B6%E6%B5%81" class="mw-redirect" title="控制流">控制流</a>
<ul><li><b><a href="/wiki/%E5%88%86%E6%94%AF_(%E8%A8%88%E7%AE%97%E6%A9%9F%E7%A7%91%E5%AD%B8)" title="分支 (计算机科学)">分支</a></b>，跳跃至程序某地址并执行相应指令；</li>
<li><b><a href="/wiki/%E6%9D%A1%E4%BB%B6%E8%A1%A8%E8%BE%BE%E5%BC%8F" class="mw-redirect" title="条件表达式">条件分支</a></b>，假设某一条件成立，就跳到程序的另一个位置；</li>
<li><b><a href="/w/index.php?title=%E9%96%93%E6%8E%A5%E5%88%86%E6%94%AF&amp;action=edit&amp;redlink=1" class="new" title="间接分支（页面不存在）">间接分支</a></b>，在跳到另一个位置之前，将现在所执行的指令的下一个指令的位置存储起来，作为<a href="/wiki/%E5%AD%90%E7%A8%8B%E5%BC%8F" class="mw-redirect" title="子程序">子程序</a>执行完返回的地址；</li></ul></li></ul>
<h3><span id=".E8.A4.87.E9.9B.9C.E6.8C.87.E4.BB.A4"></span><span class="mw-headline" id="複雜指令">复杂指令</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=4" title="编辑章节：复杂指令"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>一些电脑在他们的指令集架构内包含复杂指令。复杂指令包含：
</p>
<ul><li>将许多寄存器存成堆栈的形式。</li>
<li>移动存储器内大笔的资料。</li>
<li>复杂或是浮点数运算（<a href="/wiki/%E6%AD%A3%E5%BC%A6" title="正弦">正弦</a>，<a href="/wiki/%E9%A4%98%E5%BC%A6" title="余弦">余弦</a>，<a href="/wiki/%E5%B9%B3%E6%96%B9%E6%A0%B9" title="平方根">平方根</a>等等）</li>
<li>执行<a href="/wiki/Test-and-set" class="mw-redirect" title="Test-and-set">test-and-set</a>指令。</li>
<li>执行数字存在存储器而非寄存器的运算</li></ul>
<p>有一种复杂指令<a href="/wiki/%E5%96%AE%E6%8C%87%E4%BB%A4%E6%B5%81%E5%A4%9A%E8%B3%87%E6%96%99%E6%B5%81" class="mw-redirect" title="单指令流多资料流">单指令流多资料流</a>（<span lang="en">SIMD</span>），或<a href="/wiki/%E5%B9%B6%E8%A1%8C%E5%90%91%E9%87%8F%E5%A4%84%E7%90%86%E6%9C%BA" class="mw-redirect" title="并行向量处理机">向量指令</a>，这是一种可以在同一时间对多笔资料进行相同运算的操作。<span lang="en">SIMD</span>有能力在短时间内将大笔的向量和矩阵计算完成。<span lang="en">SIMD</span>指令使<a href="/wiki/%E4%B8%A6%E8%A1%8C%E8%A8%88%E7%AE%97" class="mw-redirect" title="并发计算">并行计算</a>变得简单，各种<span lang="en">SIMD</span>指令集被开发出来，例如<a href="/wiki/MMX" title="MMX">MMX</a>，<a href="/wiki/3DNow!" title="3DNow!">3DNow!</a>以及<a href="/wiki/AltiVec" title="AltiVec">AltiVec</a>。
</p>
<h3><span id=".E6.8C.87.E4.BB.A4.E7.9A.84.E7.B5.84.E6.88.90"></span><span class="mw-headline" id="指令的組成">指令的组成</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=5" title="编辑章节：指令的组成"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<figure typeof="mw:File/Thumb"><a href="/wiki/File:Mips32_addi.svg" class="mw-file-description"><img src="//upload.wikimedia.org/wikipedia/commons/thumb/2/2a/Mips32_addi.svg/320px-Mips32_addi.svg.png" decoding="async" width="320" height="115" class="mw-file-element" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/2/2a/Mips32_addi.svg/480px-Mips32_addi.svg.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/2/2a/Mips32_addi.svg/640px-Mips32_addi.svg.png 2x" data-file-width="500" data-file-height="180" /></a><figcaption>一条指令往往有好几个区块，这些区块包含要做的运算（加或减），可能还包含资料的原始或是目的地位置，以及常量。图中的MIPS“Add Immediate”指令允许用户选择哪个寄存器是数据源以及哪一个寄存器是要存运算后的结果，还有一个常量</figcaption></figure>
<p>在传统的架构上，一条指令包含<a href="/w/index.php?title=Op_code&amp;action=edit&amp;redlink=1" class="new" title="Op code（页面不存在）">op code</a>，表示运算的方式，以及零个或是更多的<a href="/wiki/%E9%81%8B%E7%AE%97%E5%85%83" class="mw-redirect" title="操作数">操作数</a>，有些像是操作数的数字可能指的是<a href="/wiki/%E6%9A%AB%E5%AD%98%E5%99%A8" class="mw-redirect" title="寄存器">寄存器</a>的编号，还有存储器位置，或是文字资料。
</p><p>在<a href="/wiki/%E8%B6%85%E9%95%BF%E6%8C%87%E4%BB%A4%E5%AD%97" title="超长指令字">超长指令字</a>（VLIW）的结构中，包含了许多<a href="/wiki/%E5%BE%AE%E6%8C%87%E4%BB%A4" title="微指令">微指令</a>，借此将复杂的指令分解为简单的指令。
</p>
<h3><span id=".E6.8C.87.E4.BB.A4.E7.9A.84.E9.95.B7.E5.BA.A6"></span><span class="mw-headline" id="指令的長度">指令的长度</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=6" title="编辑章节：指令的长度"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>指令长度的范围可以说是相当广泛，从<a href="/wiki/%E5%BE%AE%E6%8E%A7%E5%88%B6%E5%99%A8" class="mw-redirect" title="微控制器">微控制器</a>的4 bit，到<a href="/wiki/%E8%B6%85%E9%95%BF%E6%8C%87%E4%BB%A4%E5%AD%97" title="超长指令字">VLIW</a>系统的数百bit。在<a href="/wiki/%E5%80%8B%E4%BA%BA%E9%9B%BB%E8%85%A6" class="mw-redirect" title="个人电脑">个人电脑</a>，<a href="/wiki/%E5%A4%A7%E5%9E%8B%E9%9B%BB%E8%85%A6" class="mw-redirect" title="大型机">大型机</a>，<a href="/wiki/%E8%B6%85%E7%B4%9A%E9%9B%BB%E8%85%A6" class="mw-redirect" title="超级计算机">超级计算机</a>内的处理器，其内部的指令长度介于8到64 bits（在x86处理器结构内，最长的指令长达15 bytes，等于120 bits）。在一个指令集架构内，不同的指令可能会有不同长度。在一些结构，特别是大部分的<a href="/wiki/%E7%B2%BE%E7%AE%80%E6%8C%87%E4%BB%A4%E9%9B%86" class="mw-redirect" title="精简指令集">精简指令集</a>（RISC），指令是固定的长度，长度对应到结构内一个<a href="/wiki/%E5%AD%97" class="mw-disambig" title="字">字</a>的大小。在其他结构，长度则是<a href="/wiki/Byte" class="mw-redirect" title="Byte">byte</a>的整数倍或是一个<a href="/wiki/%E5%AD%97" class="mw-disambig" title="字">halfword</a>。
</p>
<h3><span id=".E8.A8.AD.E8.A8.88"></span><span class="mw-headline" id="設計">设计</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=7" title="编辑章节：设计"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>对微处理器而言有两种指令集。第一种是<a href="/wiki/%E8%A4%87%E9%9B%9C%E6%8C%87%E4%BB%A4%E9%9B%86" class="mw-redirect" title="复杂指令集">复杂指令集</a>（Complex Instruction Set Computer），拥有许多不同的指令。在1970年代，许多机构，像是IBM，发现有许多指令是不需要的。结果就产生了<a href="/wiki/%E7%B2%BE%E7%AE%80%E6%8C%87%E4%BB%A4%E9%9B%86" class="mw-redirect" title="精简指令集">精简指令集</a>（Reduced Instruction Set Computer），它所包含的指令就比较少。精简的指令集可以提供比较高的速度，使处理器的尺寸缩小，以及较少的电力损耗。然而，比较复杂的指令集较容易使工作更完善，存储器及<a href="/wiki/%E7%BC%93%E5%AD%98" title="缓存">缓存</a>的效率较高，以及较为简单的代码。
</p><p>一些指令集保留了一个或多个的opcode，以执行<a href="/wiki/%E7%B3%BB%E7%BB%9F%E8%B0%83%E7%94%A8" title="系统调用">系统调用</a>或<a href="/wiki/%E4%B8%AD%E6%96%B7" title="中断">软件中断</a>。
</p>
<h2><span id=".E6.8C.87.E4.BB.A4.E9.9B.86.E7.9A.84.E5.AF.A6.E4.BD.9C"></span><span class="mw-headline" id="指令集的實作">指令集的实现</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E6%8C%87%E4%BB%A4%E9%9B%86%E6%9E%B6%E6%A7%8B&amp;action=edit&amp;section=8" title="编辑章节：指令集的实现"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>在设计处理器内的<a href="/wiki/%E5%BE%AE%E6%9E%B6%E6%A7%8B" title="微架构">微架构</a>时，工程师使用藉电路连接的区块来架构，区块用来表示加法器，乘法器，计数器，寄存器，算术逻辑单元等等，<a href="/wiki/%E6%9A%AB%E5%AD%98%E5%99%A8%E5%82%B3%E9%81%9E%E8%AA%9E%E8%A8%80" title="寄存器传递语言">寄存器传递语言</a>通常被用来描述被解码的指令，指令是借由微架构来执行指令。
有两种基本的方法来建构<a href="/wiki/%E6%8E%A7%E5%88%B6%E5%8D%95%E5%85%83" class="mw-disambig" title="控制单元">控制单元</a>，藉控制单元，以微架构作为通路来执行指令：
</p>
<ol><li>早期的电脑和采用精简指令集的电脑藉将电路接线（像是微架构剩下的部分）。</li>
<li>其他的设备使用<a href="/wiki/%E5%BE%AE%E7%A8%8B%E5%BA%8F" title="微程序">微程序</a>来达成—像是晶体管ROM或PLA（即使RAM已使用很久）。</li></ol>
<p>电脑<a href="/wiki/%E5%BE%AE%E5%A4%84%E7%90%86%E5%99%A8" title="微处理器">微处理器</a>的<b>指令集架构</b>（Instruction Set Architecture）常见的有三种：
</p>
<ul><li><b><a href="/wiki/%E8%A4%87%E9%9B%9C%E6%8C%87%E4%BB%A4%E9%9B%86" class="mw-redirect" title="复杂指令集">复杂指令集运算</a></b>（Complex Instruction Set Computing，CISC）</li></ul>
<dl><dd>目前<a href="/wiki/X86" title="X86">x86</a>架构微处理器如<a href="/wiki/Intel" class="mw-redirect" title="Intel">Intel</a>的<a href="/wiki/Pentium" class="mw-redirect" title="Pentium">Pentium</a>/<a href="/wiki/Celeron" class="mw-redirect" title="Celeron">Celeron</a>/<a href="/wiki/Xeon" class="mw-redirect" title="Xeon">Xeon</a>与<a href="/wiki/AMD" class="mw-redirect" title="AMD">AMD</a>的<a href="/wiki/Athlon" class="mw-redirect" title="Athlon">Athlon</a>/<a href="/wiki/Duron" class="mw-redirect" title="Duron">Duron</a>/<a href="/wiki/Sempron" class="mw-redirect" title="Sempron">Sempron</a>；以及其64位扩展系统的<a href="/wiki/X86-64" title="X86-64">x86-64</a>架构的Intel 64的<a href="/wiki/Intel_Core" class="mw-redirect" title="Intel Core">Intel Core</a>/<a href="/wiki/Core_2" class="mw-redirect" title="Core 2">Core 2</a>/<a href="/wiki/Celeron" class="mw-redirect" title="Celeron">Celeron</a>/<a href="/wiki/Pentium" class="mw-redirect" title="Pentium">Pentium</a>/<a href="/wiki/Xeon" class="mw-redirect" title="Xeon">Xeon</a>与AMD64的<a href="/wiki/Phenom_II" class="mw-redirect" title="Phenom II">Phenom II</a>/<a href="/wiki/Phenom" class="mw-redirect" title="Phenom">Phenom</a>/<a href="/wiki/Athlon_64" class="mw-redirect" title="Athlon 64">Athlon 64</a>/<a href="/wiki/Athlon_II" class="mw-redirect" title="Athlon II">Athlon II</a>/<a href="/wiki/Opteron" title="Opteron">Opteron</a>/<a href="/wiki/AMD_APU" class="mw-redirect" title="AMD APU">AMD APU</a>/<a href="/wiki/Ryzen" class="mw-redirect" title="Ryzen">Ryzen</a>/<a href="/wiki/EPYC" class="mw-redirect" title="EPYC">EPYC</a>都属于复杂指令集。主要针对的操作系统是<a href="/wiki/%E5%BE%AE%E8%BD%AF" title="微软">微软</a>的<a href="/wiki/Windows" class="mw-redirect" title="Windows">Windows</a>和<a href="/wiki/%E8%8B%B9%E6%9E%9C%E5%85%AC%E5%8F%B8" class="mw-redirect" title="苹果公司">苹果公司</a>的<a href="/wiki/MacOS" title="MacOS">macOS</a>。另外<a href="/wiki/Linux" title="Linux">Linux</a>，一些UNIX等，都可以运行在x86（复杂指令集）架构的微处理器。</dd></dl>
<ul><li><b><a href="/wiki/%E7%B2%BE%E7%AE%80%E6%8C%87%E4%BB%A4%E9%9B%86" class="mw-redirect" title="精简指令集">精简指令集运算</a></b>（Reduced Instruction Set Computing，RISC）</li></ul>
<dl><dd>这种指令集运算包括惠普的PA-RISC，<a href="/wiki/%E5%9B%BD%E9%99%85%E5%95%86%E4%B8%9A%E6%9C%BA%E5%99%A8" class="mw-redirect" title="国际商业机器">国际商业机器</a>的<a href="/wiki/PowerPC" title="PowerPC">PowerPC</a>，<a href="/wiki/%E5%BA%B7%E6%9F%8F" title="康柏">康柏</a>（后被<a href="/wiki/%E6%83%A0%E6%99%AE" title="惠普">惠普</a>收购）的Alpha，<a href="/wiki/%E7%BE%8E%E6%99%AE%E6%80%9D%E7%A7%91%E6%8A%80" title="美普思科技">美普思科技</a>公司的MIPS，SUN公司的SPARC，ARM公司的<a href="/wiki/ARM%E6%9E%B6%E6%A7%8B" title="ARM架构">ARM架构</a>等。目前有UNIX、Linux以及包括iOS、Android、Windows Phone等在内的大多数移动操作系统运行在精简指令集的处理器上。</dd></dl>
<ul><li><b><a href="/wiki/%E9%A1%AF%E5%BC%8F%E4%B8%A6%E8%A1%8C%E6%8C%87%E4%BB%A4%E9%81%8B%E7%AE%97" title="显式并发指令运算">显式并发指令运算</a></b>（Explicitly Parallel Instruction Computing，EPIC）</li></ul>
<dl><dd>显式并发指令运算乃先进的全新指令集运算，只有Intel的<a href="/wiki/IA-64" title="IA-64">IA-64</a>架构的纯64位微处理器的<a href="/wiki/Itanium" class="mw-redirect" title="Itanium">Itanium</a>/<a href="/wiki/Itanium_2" class="mw-redirect" title="Itanium 2">Itanium 2</a>。EPIC指令集运算的IA-64架构主要针对的操作系统是微软64位安腾版的<a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>以及64位安腾版的<a href="/wiki/Windows_Server_2003" title="Windows Server 2003">Windows Server 2003</a>。另外一些64位的Linux，一些64位的UNIX也可以运行IA-64（显式并发指令运算）架构。</dd></dl>
<ul><li><b><a href="/wiki/VLIW" class="mw-redirect" title="VLIW">超长指令字指令集运算</a></b>（VLIW）</li></ul>
<dl><dd>通过将多条指令放入一个指令字，有效的提高了CPU各个计算功能部件的利用效率，提高了程序的性能</dd></dl>
