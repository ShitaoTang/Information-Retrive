<p><b>设备驱动程序</b>（英语：<span lang="en"><b>device driver</b></span>），简称<b>驱动程序</b>（<span lang="en">driver</span>），是一个允许高阶（<span lang="en">High level</span>）电脑<a href="/wiki/%E8%BB%9F%E9%AB%94" class="mw-redirect" title="软件">软件</a>（computer software）与<a href="/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%A1%AC%E4%BB%B6" title="计算机硬件">硬件</a>（hardware）交互的<a href="/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%A8%8B%E5%BA%8F" title="计算机程序">程序</a>，这种程序建立了一个硬件与硬件，或硬件与软件沟通的接口，经由主板上的<a href="/wiki/%E5%8C%AF%E6%B5%81%E6%8E%92" class="mw-redirect mw-disambig" title="总线">总线</a>（bus）或其它沟通子系统（subsystem）与硬件形成连接的机制，这样的机制使得硬件设备（device）上的资料交换成为可能。
</p><p>依据不同的计算机体系结构与操作系统差异平台，驱动程序经历了8位（8-bit）、16位（16-bit）、32位（32-bit）、64位（64-bit）变迁，这是为了调和操作系统与驱动程序之间的依存关系，例如在<a href="/wiki/Windows_3.11" class="mw-redirect" title="Windows 3.11">Windows 3.11</a>的16位操作系统时代，大部分的驱动程序都是16位，到了32位的<a href="/wiki/Windows_XP" title="Windows XP">Windows XP</a>则使用32位驱动程序（微软提供了<a href="/wiki/Windows_Driver_Model" title="Windows Driver Model">Windows Driver Model</a>可实现driver），至于64位的<a href="/wiki/Linux" title="Linux">Linux</a>或是<a href="/wiki/Windows" class="mw-redirect" title="Windows">Windows</a>平台上，就必须使用64位的驱动程序（WDM与WDF皆可实现64位驱动程序）。
</p>

<h2><span id=".E9.A9.85.E5.8B.95.E7.A8.8B.E5.BC.8F.E7.9A.84.E9.96.8B.E7.99.BC"></span><span class="mw-headline" id="驅動程式的開發">驱动程序的开发</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E9%A9%B1%E5%8A%A8%E7%A8%8B%E5%BA%8F&amp;action=edit&amp;section=1" title="编辑章节：驱动程序的开发"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>驱动程序的开发工作是很具挑战性的，因为必须配合着硬件与软件上相当明确与高级的平台技术。由于大多数的驱动程序（device drivers）执行在核心模式（kernel mode），软件的错误经常造成系统严重的不稳定，例如<a href="/wiki/%E8%97%8D%E5%B1%8F" class="mw-redirect" title="蓝屏">蓝屏</a>（blue screen），这跟过去的用户模式（user mode）下的程序设计（例如<a href="/wiki/Delphi" title="Delphi">Delphi</a>、<a href="/wiki/VB" class="mw-redirect" title="VB">VB</a>、<a href="/wiki/Java" title="Java">Java</a>）有明显的差异性。
</p>
<h3><span id="Windows.E5.B9.B3.E5.8F.B0"></span><span class="mw-headline" id="Windows平台">Windows平台</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E9%A9%B1%E5%8A%A8%E7%A8%8B%E5%BA%8F&amp;action=edit&amp;section=2" title="编辑章节：Windows平台"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p>为了大量减轻驱动程序开发人员的负担，微软不断的改进驱动程序的开发软件与架构，从早期复杂深晦的VxD，到Windows XP上的<a href="/wiki/Windows_Driver_Model" title="Windows Driver Model">Windows Driver Model</a>（以下简称WDM）开发架构，如今<a href="/wiki/Windows_Driver_Foundation" class="mw-redirect" title="Windows Driver Foundation">Windows Driver Foundation</a>（以下简称WDF）已成为新一代的Windows平台驱动程序发展架构，这个架构大量简化了驱动程序的开发流程，更符合面向对象的精神，此架构包含了User Mode Driver Framework与Kernel Mode Driver Framework两种开发模式。在开发Windows平台上的驱动程序之前，必须先安装DDK（Driver Development Kit），DDK包同时支持WDM与WDF两种架构。
</p>
<h3><span id="Linux.E5.B9.B3.E5.8F.B0"></span><span class="mw-headline" id="Linux平台">Linux平台</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E9%A9%B1%E5%8A%A8%E7%A8%8B%E5%BA%8F&amp;action=edit&amp;section=3" title="编辑章节：Linux平台"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h3>
<p><a href="/wiki/Linux" title="Linux">Linux</a>下的设备有三种基本类型：字符设备，块设备和网络接口。
</p>
<h2><span id=".E9.A9.85.E5.8B.95.E7.A8.8B.E5.BC.8F.E7.9A.84.E6.87.89.E7.94.A8"></span><span class="mw-headline" id="驅動程式的應用">驱动程序的应用</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E9%A9%B1%E5%8A%A8%E7%A8%8B%E5%BA%8F&amp;action=edit&amp;section=4" title="编辑章节：驱动程序的应用"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>因为常见的硬件和操作系统的差异性，驱动程序存在着不同的方式。用于：
</p>
<ul><li><a href="/wiki/%E6%89%93%E5%8D%B0%E6%9C%BA" title="打印机">打印机</a></li>
<li><a href="/wiki/%E6%98%BE%E5%8D%A1" class="mw-redirect" title="显卡">显卡</a></li>
<li><a href="/wiki/%E7%BD%91%E5%8D%A1" title="网卡">网卡</a></li>
<li><a href="/wiki/%E5%A3%B0%E5%8D%A1" title="声卡">声卡</a></li>
<li>不同种类的<a href="/wiki/%E6%80%BB%E7%BA%BF" title="总线">总线</a></li>
<li>不同种类的低<a href="/wiki/%E5%B8%A6%E5%AE%BD" title="带宽">带宽</a>的<a href="/w/index.php?title=%E8%BE%93%E5%85%A5/%E8%BE%93%E5%87%BA&amp;action=edit&amp;redlink=1" class="new" title="输入/输出（页面不存在）">输入/输出</a>总线（如<a href="/wiki/%E5%AE%9A%E7%82%B9%E8%AE%BE%E5%A4%87" title="定点设备">定点设备</a>：<a href="/wiki/%E9%BC%A0%E6%A0%87" title="鼠标">鼠标</a>、<a href="/wiki/%E9%94%AE%E7%9B%98" class="mw-redirect" title="键盘">键盘</a>、<a href="/wiki/USB" title="USB">USB</a>等等。）</li></ul>
<ul><li><a href="/wiki/%E7%A1%AC%E7%9B%98" title="硬盘">硬盘</a>的连接接口/RAID（<a href="/wiki/RAID" title="RAID">RAID</a>，<a href="/wiki/SCSI" class="mw-redirect" title="SCSI">SCSI</a>）;</li>
<li>实现对不同的<a href="/wiki/%E6%96%87%E4%BB%B6%E7%B3%BB%E7%BB%9F" title="文件系统">文件系统</a>的支持。</li>
<li>实现对<a href="/wiki/%E6%89%AB%E6%8F%8F%E4%BB%AA" class="mw-redirect" title="扫描仪">扫描仪</a>、<a href="/wiki/%E6%95%B0%E7%A0%81%E7%9B%B8%E6%9C%BA" title="数字相机">数字相机</a>的支持。</li></ul>
<p>驱动程序的抽象化共同层级（Common levels of abstraction）：
</p>
<ul><li>在硬件层方面：
<ul><li>直接连接（Interfacing directly）</li>
<li>使用一些较高层次的接口（如：视频<a href="/wiki/BIOS" title="BIOS">BIOS</a>）</li>
<li>使用另一低级别的驱动程序（如文件系统驱动程序使用磁盘驱动程序）</li>
<li>模拟硬件工作</li></ul></li>
<li>在软件层方面：
<ul><li>允许操作系统直接访问硬件资源</li>
<li>仅实现其原始样式（<a href="/w/index.php?title=Primitive_(computer_science)&amp;action=edit&amp;redlink=1" class="new" title="Primitive (computer science)（页面不存在）">primitive</a>s）</li>
<li>实现对非驱动程序的软件的界面（如<a href="/wiki/TWAIN" title="TWAIN">TWAIN</a>）</li>
<li>实现一个更高层级的语言，例如<a href="/wiki/PostScript" title="PostScript">PostScript</a></li></ul></li></ul>
<h2><span id=".E9.A9.85.E5.8B.95.E7.A8.8B.E5.BC.8F.E7.9A.84API"></span><span class="mw-headline" id="驅動程式的API">驱动程序的API</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E9%A9%B1%E5%8A%A8%E7%A8%8B%E5%BA%8F&amp;action=edit&amp;section=5" title="编辑章节：驱动程序的API"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<ul><li>Advanced Linux Sound Architecture（ALSA） - 标准新式Linux声卡驱动程序接口（sound driver interface）</li>
<li>I/O Kit - 一种苹果电脑（Apple）上的开放式接口适用于发展<a href="/wiki/%E9%BA%A5%E9%87%91%E5%A1%94" title="麦金塔">麦金塔</a>操作系统（Mac OS X）的设备驱动程序</li>
<li>Installable File System（IFS） - 一种IBM OS/2与Microsoft Windows NT的文件系统API</li>
<li>Network Driver Interface Specification（NDIS） - 一种标准网卡（network card）驱动程序API</li>
<li>Open Data-Link Interface（ODI） - 一种类似NDIS的网卡API</li>
<li>Scanner Access Now Easy（SANE） - a public domain interface to raster image scanner hardware</li>
<li>Uniform Driver Interface（UDI） - 跨平台（cross platform）驱动程序的接口规范</li>
<li>Windows Display Driver Model（WDDM） - Windows Vista及以后平台上新式的图形显示（graphic display）驱动程序架构。从Windows 8开始，Windows不支持旧的XPDM。</li>
<li><a href="/wiki/Windows_Driver_Foundation" class="mw-redirect" title="Windows Driver Foundation">Windows Driver Foundation</a>（WDF）</li>
<li><a href="/wiki/Windows_Driver_Model" title="Windows Driver Model">Windows Driver Model</a>（WDM）</li>
<li><a href="/w/index.php?title=Universal_Windows_Driver&amp;action=edit&amp;redlink=1" class="new" title="Universal Windows Driver（页面不存在）">Universal Windows Driver</a>（UWD）</li></ul>
