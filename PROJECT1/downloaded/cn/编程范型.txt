<p><b>编程范型</b>、<b>编程范式</b>或<b>程序设计法</b>（英语：<span lang="en">Programming paradigm</span>），是指<a href="/wiki/%E8%BD%AF%E4%BB%B6%E5%B7%A5%E7%A8%8B" title="软件工程">软件工程</a>中的一类典型的编程风格。常见的编程范型有：<a href="/wiki/%E5%87%BD%E6%95%B0%E5%BC%8F%E7%BC%96%E7%A8%8B" title="函数式编程">函数式编程</a>、<a href="/wiki/%E6%8C%87%E4%BB%A4%E5%BC%8F%E7%B7%A8%E7%A8%8B" title="指令式编程">指令式编程</a>、<a href="/wiki/%E8%BF%87%E7%A8%8B%E5%BC%8F%E7%BC%96%E7%A8%8B" title="过程式编程">过程式编程</a>、<a href="/wiki/%E9%9D%A2%E5%90%91%E5%AF%B9%E8%B1%A1%E7%A8%8B%E5%BA%8F%E8%AE%BE%E8%AE%A1" title="面向对象程序设计">面向对象编程</a>等等。
</p><p>编程范型提供并决定了<a href="/wiki/%E7%A8%8B%E5%BA%8F%E5%91%98" title="程序员">程序员</a>对<a href="/wiki/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%A8%8B%E5%BA%8F" title="计算机程序">程序</a>执行的看法。例如，在面向对象编程中，程序员认为程序是一系列相互作用的对象，由于方法论的不同，面向对象编程又分为<a href="/wiki/%E5%9F%BA%E4%BA%8E%E7%B1%BB%E7%BC%96%E7%A8%8B" title="基于类编程">基于类编程</a>和<a href="/wiki/%E5%9F%BA%E4%BA%8E%E5%8E%9F%E5%9E%8B%E7%BC%96%E7%A8%8B" title="基于原型编程">基于原型编程</a>，而在函数式编程中一个程序会被看作是一个无状态的函数计算的序列。
</p>
<h2><span id=".E6.A6.82.E8.BF.B0"></span><span class="mw-headline" id="概述">概述</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=%E7%BC%96%E7%A8%8B%E8%8C%83%E5%9E%8B&amp;action=edit&amp;section=1" title="编辑章节：概述"><span>编辑</span></a><span class="mw-editsection-bracket">]</span></span></h2>
<p>正如软件工程中不同的群体会提倡不同的“方法学”一样，不同的<a href="/wiki/%E7%BC%96%E7%A8%8B%E8%AF%AD%E8%A8%80" title="编程语言">编程语言</a>也会提倡不同的“编程范型”。一些语言是专门为某个特定的范型设计的，如 <a href="/wiki/Smalltalk" title="Smalltalk">Smalltalk</a> 和 <a href="/wiki/Java" title="Java">Java</a> 支持面向对象编程，而 <a href="/wiki/Haskell" title="Haskell">Haskell</a> 和 <a href="/wiki/Scheme" title="Scheme">Scheme</a> 则支持函数式编程，同时还有另一些语言支持多种范型，如 <a href="/wiki/Ruby" title="Ruby">Ruby</a>、<a href="/wiki/Common_Lisp" title="Common Lisp">Common Lisp</a>、<a href="/wiki/Python" title="Python">Python</a>、<a href="/wiki/Rust" title="Rust">Rust</a>。
</p><p>很多编程范型已经被熟知他们禁止使用哪些技术，同时允许使用哪些。例如，<a href="/wiki/%E7%BA%AF%E5%87%BD%E6%95%B0%E5%BC%8F%E7%BC%96%E7%A8%8B" title="纯函数式编程">纯函数式编程</a>不允许有副作用<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup>；大部分高阶程序语言都期望用户进行<a href="/wiki/%E7%BB%93%E6%9E%84%E5%8C%96%E7%BC%96%E7%A8%8B" title="结构化编程">结构化编程</a>而避免<a href="/wiki/%E9%9D%9E%E7%B5%90%E6%A7%8B%E5%8C%96%E7%A8%8B%E5%BC%8F%E8%A8%AD%E8%A8%88" title="非结构化编程">非结构化编程</a>，其中结构化编程不允许使用 <a href="/wiki/Goto" title="Goto">goto</a>。可能是因为这个原因，新的范型常常被那些习惯于较早的风格的人认为是教条主义或过分严格。然而，这样避免某些技术反而更加证明了关于程序正确性，或仅仅是理解它的行为的法则，而不用限制程序语言的一般性。
</p><p>编程范型和编程语言之间的关系可能十分复杂，由于一个编程语言可以支持<a href="/wiki/%E5%A4%9A%E9%87%8D%E7%BC%96%E7%A8%8B%E8%8C%83%E5%9E%8B" title="多重编程范型">多种范型</a>。例如，<a href="/wiki/C%2B%2B" title="C++">C++</a> 设计时，支持过程式编程、面向对象编程以及<a href="/wiki/%E6%B3%9B%E5%9E%8B%E7%BC%96%E7%A8%8B" title="泛型编程">泛型编程</a>。然而，设计师和程序员们要考虑如何使用这些范型元素来构建一个程序。一个人可以用 C++ 写出一个完全过程化的程序，另一个人也可以用 C++ 写出一个纯粹的面向对象程序，甚至还有人可以写出杂揉了两种范型的程序。
</p>
